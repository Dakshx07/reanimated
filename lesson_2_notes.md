# 📝 Lesson 2 Study Notes: The Worklet Compiler & The Closure Capture Trap

This document details the compile-time and runtime behavior of JavaScript Worklets inside React Native.

---

## 🛠️ 1. What are Babel and Metro?

When building a React Native application, your code passes through two critical build tools before it executes on a device:

1. **Babel (The Compiler)**:
   * **Role**: Babel takes modern JavaScript/TypeScript (and JSX) and compiles (translates) it into standard, widely-compatible JavaScript.
   * **Reanimated Role**: Reanimated uses a Babel plugin. When Babel sees the `"worklet";` directive inside a function, it intercepts it and rewrites the function into a structured **Worklet Object** configuration.
2. **Metro (The Bundler)**:
   * **Role**: Metro is the React Native code packager. It takes hundreds of separate files and assets in your codebase and packages (bundles) them into a single file (`index.bundle`) that the Hermes JS engine can execute.

---

## ⚙️ 2. What is a Worklet & Worklet Object?

* **Worklet**: A JavaScript function compiled specifically to run on a secondary JavaScript engine instance (runtime) running on another thread (like the UI thread's render loop).
* **Worklet Object**: The JavaScript representation of a compiled worklet function generated by the Babel compiler. It contains the following properties:
  1. `__workletHash`: A unique number identification. When the app boots, this is registered in the C++ global **Call Table** so C++ knows how to execute it on the UI thread.
  2. `code`: The JavaScript function compiled as a string, ready to be sent and run on the UI Thread VM.
  3. `__closure`: A dictionary containing copies of all outer-scope variables that the function needs to run.

---

## 🧠 3. What is a Closure in JavaScript?

### In Standard JavaScript:
A **closure** is a feature where an inner function "remembers" and has access to variables in its outer (parent) scope, even after the parent function has finished executing.

```javascript
function makeCounter() {
  let count = 0; // Outer variable
  
  return function() { // Inner function (Closure)
    count++; // count is captured!
    console.log(count);
  };
}

const counter = makeCounter();
counter(); // Prints: 1
counter(); // Prints: 2
```

In standard JavaScript, the closure holds a **live memory reference** to `count`. If `count` changes in the outer scope, the inner function sees it instantly because they share the same memory heap.

### ⚠️ In Worklets (The Capture Trap):
Because the JS VM and the UI VM run on separate threads with **isolated memory heaps**, a worklet cannot share a reference pointer to an outer-scope JS variable.
* Instead, the compiler **deeply clones (copies by value)** the variable into the Worklet Object's `__closure` dictionary.
* Once copied, the connection is broken. Changing the variable on the JS thread will **never** update the copy inside the UI thread's VM.

## ❓ Lesson 2 Q&A

### Q1: Why does a closure in a worklet print the old value of a JS variable even if that variable was updated on the JS thread?
> **Answer**: Because the JS VM and UI VM do not share the same memory heap. Variables captured from the outer scope are copied (cloned) into the worklet's `__closure` property. Since the worklet holds a copy by value instead of a reference pointer, updates on the JS Thread VM are not reflected in the UI Thread VM.

### Q2: If you pass a massive array (`[1, 2, ... 100000]`) as a closure variable inside a worklet, what happens under the hood, and how does it affect performance?
> **Answer**: Under the hood, the Worklets engine will deeply clone the entire array from the JS VM's memory heap into the UI VM's memory heap. This is a heavy serialization/deserialization CPU operation that can cause major frame drops on the JS thread during the cloning phase and increase memory garbage collection pressure.

### Q3: What three main properties are inside a compiled Worklet Object that Metro/Babel generates?
> **Answer**:
> 1. `__workletHash`: A unique hash identification for C++ Call Table registration.
> 2. `code`: The compiled JS function code as a string to be run inside the UI VM.
> 3. `__closure`: A dictionary containing copies of all outer-scope variables captured by the function.
